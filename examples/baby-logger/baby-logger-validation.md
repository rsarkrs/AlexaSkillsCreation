# Baby Logger Validation

## Design Validation

Status: valid for a v1 guided implementation.

What holds:

- The invocation name `baby tracker` is more reliable for physical-device routing than `baby logger`.
- Splitting direct logs from timer start and stop flows is the right interaction boundary.
- The slot model stays simple and avoids fragile free-form capture.
- The Google Apps Script backend pattern fits the logging use case and the repository guidance.
- The hosted Alexa Node adapter in front of Apps Script is the correct integration pattern for this repo.

## Risky Assumptions

1. The design spec mixes `mixed diaper` in user phrases with `both` in the backend contract. The interaction model needs one normalized value strategy.
2. The design says timestamps should preferably be generated by the backend, but the sample payloads require Alexa-side timestamps. The implementation should treat request timestamps as optional and canonicalize server-side.
3. The backend contract mentions an optional auth header, but Google Apps Script web apps do not provide a reliable custom-header pattern for `doPost(e)`. A shared secret form field is safer for v1.
4. Timer state is modeled as a single active record per timer type. That is acceptable for one household stream, but it assumes no concurrent breastfeeding or tummy-time sessions need to exist.
5. The design spec says `hosted Lambda or equivalent backend adapter`, but deployment steps imply a direct Apps Script connection. The implementation must keep the Alexa-hosted Node adapter as the actual runtime endpoint.
6. `baby logger` may be misheard or poorly routed on physical devices even when the Test console works. Use `baby tracker` as the example default and validate launch before backend debugging.
7. Unit capture assumes `ounces`, `ounce`, `oz`, `milliliters`, `milliliter`, and `ml` cover the real phrasing. Alexa slot resolution should be tested with both singular and plural forms.
8. Alexa-hosted skills need direct dependencies declared in `package.json`; relying on transitive availability can cause `Cannot find module 'ask-sdk-model'`.
9. Apps Script can log data successfully while returning an intermediate redirect/non-JSON response. The adapter must follow redirects before parsing JSON.

## Decisions For Implementation

- Normalize diaper values to `pee`, `poop`, and `both`, with `mixed` treated as a synonym for `both`.
- Send explicit backend `action` names from the Alexa adapter.
- Keep timestamps in the request payload for traceability, but let the backend replace missing or malformed values with server time.
- Use a shared secret form field named `token` for Apps Script protection in v1.
- Store timer state in a `SessionState` sheet tab keyed by timer type.
- Generate artifacts for guided execution only because MCP browser automation is disabled in [workflow-config.json](../../config/workflow-config.json).
- Use `baby tracker` for the interaction model invocation name.
- Keep the redirect-aware Lambda backend caller and direct `ask-sdk-model` dependency in generated artifacts.
